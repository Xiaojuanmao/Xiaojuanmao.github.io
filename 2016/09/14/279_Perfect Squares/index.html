279. Perfect Squares

Given a positive integer n, find the least number of perfect square numbers (for example, 1, 4, 9, 16, ...) which sum to n.

For example, given n = 12, return 3 because 12 = 4 + 4 + 4; given n = 13, return 2 because 13 = 4 + 9.

思路

题目的意思也就是给出一个数字n，用最少的平方数相加得到n，返回消耗的最少的数字个数。

最开始的想法就是先找出刚好比n小的一个平方数，这个平方数是确定的第一个数字，好像有点贪婪算法的思想，每次都去找一个刚好比n小的平方数，再将n减去这个平方数，得到的结果再去按相同的方法递归寻找，直到刚好最后为0了。

后来再一想好像不对，例如输入的n为12的时候，按照上面的那套说法，就会分解成了12 = 9 + 1 + 1 + 1。可能还需要在上面的基础上再进一步改进，对于一个n来说，分解的方式有很多种，需要进行一个对比后再出最后的结果。

动态规划 (61 ms)

用DP来解决这个问题的思路，实际上是将从1到n的所有数字的最小结果都算出来，再直接返回n的结果就好。算法比较好的一点在于，只要a + b * b不是平方数，那么dp[a + b * b]最小的值必定是dp[a] + 1，因为只需要在之前算好的a的结果上再加上1，之前dp[a]的结果有最小的保障，再加上1必定是最小的。

四平方和定理(2 ms)

这个东西我也是从来都没有听说过，数学真的感觉是白学了= =，定理的内容以及一点小补充是这样的：

任意一个正整数都可以写成不超过4个完全平方数的和。
因为任何数加上0²都不变。所以对于不够4个完全平方数的可以用0²补齐
例如:5=2²+1²=2²+1²+0²+0²
于是四平方和定理等价为:任意一个正整数都可以写成4个完全平方数的和

下面有两个定理，能够帮助简化计算：

  • 优于四平方和定理，返回的结果只能是 1, 2, 3, 4里面的一个
  • 如果含有因子4，可以把4都去掉，比如2和8,3和12等等，返回的结果都相同
  • 如果一个数除以8余7的话，那么肯定是由4个完全平方数组成.

上面算法的逻辑也就是这样的：

  1. 如果n是个完全平方数或者由两个完全平方数相加得到，则直接返回1或者2
  2. 如果n能够被4整除，则n的结果和n/4返回的结果相同
  3. 如果n被8除得7，则必定由4个完全平方数组合而成
  4. 其余情况返回3
