EventBus解析(二)调用链跟踪

工作流程

  1. 由EventBus或者EventBusBuilder这两个类提供的API来注册register，也就是给eventbus这套框架指定需要检索的具有@Subscribe注解方法的类实例。
  2. 在register(Object subscriber)的过程中，用到了SubscriberMethodFinder这个工具类，这个工具类利用反射以及很多的缓存机制，寻找传入对象subscriber中符合规定的方法，并记录在一个Map<Class<?>, List<Subscription>>中，这里会按照优先级对订阅了同一event进行排序，上篇博客提到的priority就是在这时候起的作用。
  3. 当用户调用了post(MsgEvent event)方法之后，就会根据调用线程是否为主线程以及调用线程的一些相关信息，来对订阅过MsgEvent类型以及订阅过MsgEvent父类的一些方法进行任务处理。这里的任务处理指的不仅仅是简单的调用，而是根据在@Subscribe(threadMode = ThreadMode.XXXXX)注解上面的一些配置项来看的。之前提到过的四种线程模式就在这里开始起作用了，框架里面用到了三种处理方式来实现不同的效果，用到了下面这三个类：
    HandlerPoster ThreadMode.MAIN
BackgroundPoster ThreadMode.BACKGROUND
AsyncPoster ThreadMode.ASYNC
    来对不同的方法进行调度，然后就有了很方便使用的eventbus框架，一些细节部分后面结合源代码来看。

主要类型

  1. EventBus: 最基本的类，类似于事件分发管理的一个中心枢纽
  2. EventBusBuilder: 从名字能看的出，建造者模式的一个应用，用于方便的实例化不同配置的EventBus
  3. PostingThreadState: 一个用来存储不同线程线程私有数据的数据结构，包含了一些字段例如：是否为主线程 ，事件队列等等。框架中将其与ThreadLocal结合一起使用
  4. HandlerPoster: 对应ThreadMode.MAIN模式下的事件处理器
  5. BackgroundPoster: 对应ThreadMode.BACKGROUND模式下的事件处理器
  6. AsyncPoster: 对应ThreadMode.ASYNC模式下的事件处理器
  7. SubscriberMethodFinder: 用反射的方法来寻找带有@Subscribe注解的符合条件规范的回调方法

过程分析

  1. 创建EventBus实例
        一般用到的都是getDefault()，在这个里面用到了单例模式，为了防止并发导致错误，采用了类锁的机制，并且官方建议是使用getDefault()方法，因为其他创建出来的EventBus实例不是唯一的，有可能每个线程创建出来的不同实例，这时候再post消息的话就在不同的EventBus里面了。代码如下：

     public static EventBus getDefault() {
    if (defaultInstance == null) {
        synchronized (EventBus.class) {
            if (defaultInstance == null) {
                defaultInstance = new EventBus();
            }
        }
    }
    return defaultInstance;
}

  2. 注册目标类
        在外部调用eventBus.register(object)方法，就能够完成用到了SubscriberMethodFinder这个类，使用反射来找用注解标示过的符合规定的方法。寻找到的方法用SubscriberMethod类将方法包装起来，这个类里面还记录着一些其他的信息，例如：ThreadMode、eventType等等。

    public void register(Object subscriber) {
    Class<?> subscriberClass = subscriber.getClass();
    List<SubscriberMethod> subscriberMethods = subscriberMethodFinder.findSubscriberMethods(subscriberClass);
    synchronized (this) {
        for (SubscriberMethod subscriberMethod : subscriberMethods) {
            subscribe(subscriber, subscriberMethod);
        }
    }
}

        在后面详细讲解如何通过SubscriberMethodFinder来找到那些方法的，这里只需要知道现在已经找到了所有订阅方法，并获取到了它们将被调用在哪个线程、它们订阅了哪些事件等等信息，都封装在了SubscriberMethod方法里面。后面对每个SubscriberMethod都调用了subscribe()方法。
        这个方法将subscriber(register函数里面的那个object)以及subscribeMethod封装成了一个Subscription对象，并将一些信息缓存到了两个map中。

    private void subscribe(Object subscriber, SubscriberMethod subscriberMethod) {
    Class<?> eventType = subscriberMethod.eventType;
    Subscription newSubscription = new Subscription(subscriber, subscriberMethod);
    CopyOnWriteArrayList<Subscription> subscriptions = subscriptionsByEventType.get(eventType);
    if (subscriptions == null) {
        subscriptions = new CopyOnWriteArrayList<>();
        subscriptionsByEventType.put(eventType, subscriptions);
    } else {
        if (subscriptions.contains(newSubscription)) {
            throw new EventBusException("Subscriber " + subscriber.getClass() + " already registered to event "
                    + eventType);
        }
    }

    int size = subscriptions.size();
    for (int i = 0; i <= size; i++) {
        if (i == size || subscriberMethod.priority > subscriptions.get(i).subscriberMethod.priority) {
            subscriptions.add(i, newSubscription);
            break;
        }
    }

    List<Class<?>> subscribedEvents = typesBySubscriber.get(subscriber);
    if (subscribedEvents == null) {
        subscribedEvents = new ArrayList<>();
        typesBySubscriber.put(subscriber, subscribedEvents);
    }
    subscribedEvents.add(eventType);

    if (subscriberMethod.sticky) {
        if (eventInheritance) {
            // Existing sticky events of all subclasses of eventType have to be considered.
            // Note: Iterating over all events may be inefficient with lots of sticky events,
            // thus data structure should be changed to allow a more efficient lookup
            // (e.g. an additional map storing sub classes of super classes: Class -> List<Class>).
            Set<Map.Entry<Class<?>, Object>> entries = stickyEvents.entrySet();
            for (Map.Entry<Class<?>, Object> entry : entries) {
                Class<?> candidateEventType = entry.getKey();
                if (eventType.isAssignableFrom(candidateEventType)) {
                    Object stickyEvent = entry.getValue();
                    checkPostStickyEventToSubscription(newSubscription, stickyEvent);
                }
            }
        } else {
            Object stickyEvent = stickyEvents.get(eventType);
            checkPostStickyEventToSubscription(newSubscription, stickyEvent);
        }
    }
}

  3. 发送事件
        在上面两个步骤之后，特定事件的接收者相当于已经确定下来了，那些@OnSubscribe注解标注的方法以及他们能够处理的事件类型都已经被缓存到内存里面了，这个时候我们再调用post(event)方法，发送事件。

    public void post(Object event) {
    PostingThreadState postingState = currentPostingThreadState.get();
    List<Object> eventQueue = postingState.eventQueue;
    eventQueue.add(event);
    if (!postingState.isPosting) {
        postingState.isMainThread = Looper.getMainLooper() == Looper.myLooper();
        postingState.isPosting = true;
        if (postingState.canceled) {
            throw new EventBusException("Internal error. Abort state was not reset");
        }
        try {
            while (!eventQueue.isEmpty()) {
                postSingleEvent(eventQueue.remove(0), postingState);
            }
        } finally {
            postingState.isPosting = false;
            postingState.isMainThread = false;
        }
    }
}

        PostingThreadState类是配合着ThreadLocal来使用的，也就是说每个线程都有一个这样的数据结构，类里面封装了一个事件队列等信息，具体代码如下：

    final static class PostingThreadState {
    final List<Object> eventQueue = new ArrayList<Object>();
    boolean isPosting;
    boolean isMainThread;
    Subscription subscription;
    Object event;
    boolean canceled;
}

        取出了调用post()方法的线程里所存储的PostThreadState实例对象，将新来的event放了进去。后面的一部分相当于检测这个线程是否处于工作状态，如果没有工作就执行if(!postState.isPosting())里面的代码，对eventQueue里面存放的事件挨个进行postSingleEvent()处理，如果正在工作就什么也不用管了。最后处理完毕将isPosting赋值为false代表停止工作了。
        对eventQueue里面的event挨个调用postSingleEvent()方法，进入该方法后发现开始利用之前找出的那些关于Subscribe的信息，筛选出订阅了event类型事件的订阅方法。

    private void postSingleEvent(Object event, PostingThreadState postingState) throws Error {
    Class<?> eventClass = event.getClass();
    boolean subscriptionFound = false;
    if (eventInheritance) {
        List<Class<?>> eventTypes = lookupAllEventTypes(eventClass);
        int countTypes = eventTypes.size();
        for (int h = 0; h < countTypes; h++) {
            Class<?> clazz = eventTypes.get(h);
            subscriptionFound |= postSingleEventForEventType(event, postingState, clazz);
        }
    } else {
        subscriptionFound = postSingleEventForEventType(event, postingState, eventClass);
    }
    if (!subscriptionFound) {
        if (logNoSubscriberMessages) {
            Log.d(TAG, "No subscribers registered for event " + eventClass);
        }
        if (sendNoSubscriberEvent && eventClass != NoSubscriberEvent.class &&
                eventClass != SubscriberExceptionEvent.class) {
            post(new NoSubscriberEvent(this, event));
        }
    }
}

        如果在构建EventBus的时候，eventInheritance的值为true，则是允许订阅过event类型事件父类的订阅方法来处理该event类型，默认为true，否则只允许订阅event类型的订阅方法来处理。处理的方法为postSingleEventForEventType()方法，进一步调用了postToSubscription()方法。

    private void postToSubscription(Subscription subscription, Object event, boolean isMainThread) {
    switch (subscription.subscriberMethod.threadMode) {
        case POSTING:
            invokeSubscriber(subscription, event);
            break;
        case MAIN:
            if (isMainThread) {
                invokeSubscriber(subscription, event);
            } else {
                mainThreadPoster.enqueue(subscription, event);
            }
            break;
        case BACKGROUND:
            if (isMainThread) {
                backgroundPoster.enqueue(subscription, event);
            } else {
                invokeSubscriber(subscription, event);
            }
            break;
        case ASYNC:
            asyncPoster.enqueue(subscription, event);
            break;
        default:
            throw new IllegalStateException("Unknown thread mode: " + subscription.subscriberMethod.threadMode);
    }
}
