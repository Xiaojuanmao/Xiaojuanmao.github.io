WebView缓存策略

    存在两种缓存：

  1. 网页数据缓存(存储打开过的页面以及资源)
  2. H5缓存(即appcache)
网页缓存

    打开缓存的方法

webView.getSettings().setCacheMode(WebSettings.LOAD_DEFAULT);
    // 开启 DOM storage API 功能
    webView.getSettings().setDomStorageEnabled(true)

  1. 缓存构成(一个目录，两个db),可能会不同

    /data/data/package_name/cache/
/data/data/package_name/database/webview.db
/data/data/package_name/database/webviewCache.db

  2. 缓存模式

    • LOAD_DEFAULT
      根据cache-control决定是否从网络上获取数据
    • LOAD_NORMAL
      普通模式
    • LOAD_CACHE_ELSE_NETWORK
      只要本地有，无论是否过期，或者no-cache，都使用缓存中的数据。
    • LOAD_NO_CACHE
      只从网络上获取信息
    • LOAD_CACHE_ONLY
      不使用网络，只使用缓存
      如：m.taobao.com的cache-control为no-cache，在模式LOAD_DEFAULT下，无论如何都会从网络上取数据，如果没有网络，就会出现错误页面；在LOAD_CACHE_ELSE_NETWORK模式下，无论是否有网络，只要本地有缓存，都使用缓存。本地没有缓存时才从网络上获取。
m.sina.com.cn的cache-control为max

-age=60，在两种模式下都使用本地缓存数据。
总结：根据以上两种模式，建议缓存策略为，判断是否有网络，有的话，使用LOAD_DEFAULT，无网络时，使用LOAD_CACHE_ELSE_NETWORK。

  1. 清除缓存

    clearCache(boolean includeDiskFiles)
includeDiskFiles为true，则除了清除RAM里的缓存，还需要清除磁盘上的缓存文件

  2. 控制大小

    没有提供API，可以定时清除缓存大小、将缓存文件按时间顺序清除
H5缓存

  1. 缓存构成
        根据setAppCachePaych(String appCachePath)提供的路径，在H5使用缓存过程中生成缓存文件。

  2. 缓存模式
        没有模式选择，只有选择是否开启H5缓存的开关，默认为关闭的。

  3. 清除缓存
        将上面设置的缓存路径下文件清除就ok

  4. 控制大小
        可以通过setAppCacheMaxSize(long xxx)来设置缓存的最大容量，默认为Max Integer

  1. 资源文件本地存储
        通过和服务器端的约定，将一些不会变动的资源文件打包进apk里面，重写WebVWebView缓存策略
    iew的方法，检测本地存储有资源文件，就直接从本地读取。

  2. 客户端的UI优化
    在加载网页之前，将引导页面或者欢迎界面放在前面展示，WebView页面加载完毕之后再显示出WebView的布局
