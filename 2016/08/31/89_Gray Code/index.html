现在假设array(k)为一个n = k时候应该返回的序列，f(k)就是第k位不变，其余位数按照之前的规则变化的序列,也就是f(k) = (2^k-1) + array(k - 1)。那么array(k)就是array(k - 1)序列和f(k)序列的合并。

格雷码(1ms)

在做出了上面两种解法之后，好像没想出什么更好的办法了，之后去网上找了一些资料，发现好像这个问题涉及到了一个叫做格雷码的东西

在一组数的编码中，若任意两个相邻的代码只有一位二进制数不同，则称这种编码为格雷码（Gray Code），另外由于最大数与最小数之间也仅一位数不同，即“首尾相连”，因此又称循环码或反射码。

下面是格雷码生成的策略，好像在前面自己慢慢推测的时候已经把递归生成格雷码的方法推出来了，一毛一样= =。

  • 1位格雷码有两个码字
  • (n+1)位格雷码中的前2n个码字等于n位格雷码的码字，按顺序书写，加前缀0
  • (n+1)位格雷码中的后2n个码字等于n位格雷码的码字，按逆序书写，加前缀1[3]
  • n+1位格雷码的集合 = n位格雷码集合(顺序)加前缀0 + n位格雷码集合(逆序)加前缀1
