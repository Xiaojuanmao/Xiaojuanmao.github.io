Android中图片的加载

前言

    来北京之后变懒了，之前学到新的知识都会及时的总结并做好相关的记录，来了一个月没总结几次，每天的工作量并不大但人整个处于一种慵懒的状态。最近接触到了图片的加载方面一些问题，这些东西都被网上的大神们讲烂了，各种看博客学习了一段时间后，主要接触了这些关键词ImageView Matrix Fresco Scalable Bitmap,项目里面图片加载框架是Fresco,遇到了一个加载长图片的需求。

    看到了图库里面炫酷的效果之后想实现下，但是刚开始并不知道从哪里下手，后面慢慢接触，学习的历程大概分为下面这几个步骤：

  • 手动加载图片
  • 手动局部decode
  • 自定义支持可拖拽、放缩的控件
  • 结合Fresco以及自定义控件实现可放缩并自动加载图片
  • 先加载一张粗略图，根据用户手势动态decode清晰图片(还没弄出来)

1.3 坑

    在手动加载图片的时候遇到了一个坑，坑的描述如下：
W/OpenGLRenderer(10630): Bitmap too large to be uploaded into a texture (4476x885, max=4096x4096)
    在加载较大图片的时候肯定会遇到的一个问题，图片的尺寸超过了4096之后在开启了硬件加速的机器上是不允许这么大的bitmap被画在屏幕上的，所以问题就来了，怎么把这个大家伙弄到屏幕上面。我大概了解到了下面这几种解决的方法：

  1. 直接关闭硬件加速，强行把大图片直接加载并显示。这种方法太耗内存了，很容易出现什么OOM之类的问题，让图片这么放肆不太对
  2. 分块进行加载，举个长图片的例子，把一张长图片分割成很多块小图片，用很多个imageview进行分开显示再无缝拼接，android本身也提供了局部decode的方法了。但是这种方式我在想放缩要怎么办，好象不是太好处理喏。
  3. 动态局部decode，根据用户当前屏幕显示的位置来从原图片中截取可见部分显示到屏幕上

3. 自定义支持放缩、拖拽的View

    主要会用到下面一些点：

  1. Matrix : 用于对图片直接进行放缩以及平移等操作
  2. GestureDetector : 用于检测用户的单击事件以及双击事件
  3. OnTouchListener : 用于对用户的ACTION_MOVE、ACTION_DOWN、ACTION_UP等事件的判断

3.1 关于Matrix

    字面意思为矩阵，接触这个是因为其能够对Bitmap进行平移、旋转、缩放、扭曲等操作，在用户拖拽控件的时候需要对里面显示的图片进行一些操作，就靠这个Matrix，Matrix内部通过维护一个float[9]的数组来构成3x3矩阵的形式，而实际上所有的变换方法说到底就是通过更改数组中某个或某几个位置的数值，Matrix提供了setValues()和getValues()方法来操作数组。3×3数组的排列如下：

column column column
MSCALE_X (横向放缩比例) MSKEW_X （横向扭曲） MTRANS_X （横向平移）
MSKEW_Y （纵向扭曲） MSCALE_Y （纵向放缩比例） MTRANS_Y （纵向平移）
MPERSP_0 MPERSP_1 MPERSP_2

    在ImageView中使用Matrix对显示的图像进行操作，涉及到的操作如下：

平移
mMatrix.postTranslate(transX, transY);
放缩
mMatrix.postScale(tScale, tScale, fingerMidPoint.x, fingerMidPoint.y);
